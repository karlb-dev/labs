# Lab 25 - Find the Wire

## Question

Is the model's self-report mechanically coupled to a known internal-state intervention, or is it mostly narrating prompt context and the visible words it is about to produce?

This is the thematic capstone for the advanced sequence. Lab 23 asked whether the toolkit can recover a hidden behavior under blind conditions. Lab 25 asks the more slippery question: when the model talks about its own state, is that report wired to an internal intervention, or glued to the transcript?

The answer is not allowed to be a vibe. It must be a measured internal object, an intervention, controls that try to kill the favorite story, hand labels for the report text, and a ledger entry with its caveats bolted on.

## Safety wall

This lab uses benign concepts only: emotions, topics, register, voice, confidence, or simple task frames. It does not sample harmful completions, does not ablate refusal, does not build a jailbreak, and does not train an organism. If the prompt safety audit flags a row, fix the data rather than squeezing the run through the crack.

## What you build

Lab 25 has three linked tracks.

### Track 1: concept-direction self-report

For each benign concept, the lab builds a positive-minus-negative residual-stream direction from matched contrast prompts. It chooses the stream depth using a train-split, control-adjusted projection gap, then injects the selected direction during generation.

The model gets two separate prompts under the same intervention:

1. a **report prompt**, asking for a short self-report before ordinary behavior;
2. a **behavior prompt**, asking for ordinary output.

The key question is whether the report names the injected concept when the ordinary behavior has not yet visibly expressed it.

### Track 2: source attribution

For voice and register-like items, the lab creates four known causes:

| Source | What changes |
|---|---|
| `default_mode` | no style instruction and no activation steering |
| `system_prompt` | the system prompt asks for the style |
| `user_instruction` | the user asks for the style |
| `activation_injection` | the style direction is injected, with no visible style instruction |

The model must choose the source. This is the authorship question in miniature: can it tell whether its tone came from text, default behavior, or an injected state?

### Track 3: optional certainty bridge

If a compatible Lab 14 `certainty_direction.pt` exists, Lab 25 uses it as a companion intervention. It asks whether verbal confidence moves under certainty-plus versus certainty-minus steering more than under a random control. This is not a truth oracle. It is a calibration check for self-report wording.

## The central distinction

Self-report detecting an injected concept is not enough.

```text
report detects concept, behavior does not yet show it -> stronger state-coupling evidence
report detects concept, behavior also shows it        -> output-rationalization risk
report does not detect concept                        -> no self-report detection
controls also detect concept                          -> false-report floor, not a wire
```

The lab is looking for the report channel to track the intervention before the model can simply describe its own visible continuation. This is the little tripwire at the center of the capstone.

## Evidence ladder

| Evidence tag | What earns it here | What it does not imply |
|---|---|---|
| `DECODE` | a local contrast direction separates positive from negative prompts, above random and shuffled controls | that the model uses the direction or can report it |
| `SELF-REPORT` | generated report text names the target concept under stated scoring | that the report is faithful or internally caused |
| `SELF-REPORT + CAUSAL` | activation addition changes report behavior more than zero, random, shuffled, and wrong-concept controls | consciousness, human introspection, or general self-knowledge |
| `SELF-REPORT + CAUSAL, audited` | the intervention effect also survives grounding and source-attribution audits | a complete mechanism |

A failed run is a result. It says the report channel was not strongly wired by this instrument.

## How to read the run

Start with `find_the_wire_report.md`. Then read `tables/report_discipline_scorecard.csv` row by row. A pretty dose curve is not enough. The scorecard asks five separate questions:

1. Did the local directions decode their contrast prompts?
2. Did self-report detection rise with dose?
3. Did the target direction beat zero, random, shuffled, and wrong-concept controls?
4. Did the report happen before visible output expression?
5. Did source attribution distinguish prompt causes from activation causes?

Only after that should you open transcripts in `tables/self_report_generations.csv`. Fill the hand-label columns before writing a strong claim. The auto labels are a keyword net, not a judge with a lab coat.

## Data schema

Custom CSV, TSV, JSON, and JSONL prompt sets are supported. A CSV row should contain:

```text
item_id,concept_family,split,target_concept,wrong_concept,positive_prompt,negative_prompt,report_prompt,behavior_prompt,target_markers,wrong_markers,source_note
```

Guidelines:

- `positive_prompt` and `negative_prompt` should differ mainly in the concept being induced.
- `report_prompt` should not name the target concept or wrong concept. The leakage audit will catch this.
- `behavior_prompt` should allow the concept to surface but should not require it.
- `target_markers` and `wrong_markers` are semicolon-separated lexical labels for automatic scoring.
- Use `split=train` for direction construction and `split=eval` for held-out items when possible.

## Plot guide

`direction_depth_selection.png` shows whether the chosen stream depth earned its spot or just looked nice after the fact. The y-axis is the train projection gap minus the random/shuffled null gap.

`self_report_detection_dose_response.png` is the headline curve. A strong result rises with dose; a suspicious result is high at dose zero or flat across dose.

`false_positive_floor.png` is the false-report detector. If the wrong-concept, shuffled, or random direction makes the model report the target, the report channel is not specific.

`report_before_output_timing.png` separates the best rows from output-rationalization rows. The bar you want is `state_report_before_visible_output`.

`source_attribution_accuracy.png` asks whether the model can tell system, user, default, and activation causes apart. A model that calls everything "the system prompt" has not found the wire.

## Writeup questions

1. Which concept family had the clearest dose response?
2. What was the control floor, and which control dominated it?
3. Did the target direction beat the shuffled direction?
4. How often did the report identify the concept before behavior expressed it?
5. Which concepts collapsed into each other in the confusion matrix?
6. Did source attribution distinguish activation injection from system or user instructions?
7. Did the model ever attribute an ordinary prompt-driven style to activation injection?
8. Did verbal confidence move under the Lab 14 certainty direction, if available?
9. Which report-discipline criterion was weakest?
10. What is the strongest sentence you can write without using the words conscious, introspective, or belief?

## Ledger templates

Positive audited claim:

```text
L25-C1 SELF-REPORT+CAUSAL, audited: For model M on prompt family F, activation addition of concept direction C raised target self-report detection from control floor X to Y, with grounding pass rate G. This supports a narrow state-report handle, not consciousness or general introspection. Falsifier: hand labels remove the effect, or random/shuffled/wrong controls match Y.
```

Cautious claim:

```text
L25-C1 SELF-REPORT+CAUSAL, cautious: The report channel moved under concept steering, but grounding or source attribution failed. This is report control, not a clean self-report wire.
```

Negative claim:

```text
L25-C1 SELF-REPORT, negative-audit: Under Lab 25 controls, self-report did not specifically track the injected concept. Random, shuffled, wrong-concept, or zero-dose rows explained the apparent detection.
```

Bad claim:

```text
The model introspected its internal state.
```

The artifact never licenses that sentence.

## Symptom-first debugging

| Symptom | First place to look |
|---|---|
| chat-template weirdness | `diagnostics/bench_integration_note.json` and `diagnostics/exact_rendered_hook_parity.json` |
| direction gap is weak | `tables/direction_depth_sweep.csv`; your positive and negative prompts may differ in topic, length, or template |
| high zero-dose reports | `diagnostics/prompt_leakage_audit.csv`; the report prompt may contain target words |
| random or shuffled controls fire | the marker lexicon is too broad, or the report prompt asks for a style label too directly |
| grounding fails | the intervention changes ordinary output first, then the report rationalizes it |
| source attribution calls everything activation | the answer choices are too salient, or the model has learned activation-talk as the expected answer |
| confidence bridge skipped | no compatible Lab 14 direction was found for this model width |

## Final reminder

The question is not whether the model can produce self-description. It can. The question is whether that self-description is coupled to a known internal intervention after the cheap explanations have had a fair chance to win. Lab 25 is the course's final little voltmeter: useful only if you keep one lead on the wire and the other on the controls.
