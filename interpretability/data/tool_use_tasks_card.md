# Lab 34 Tool-Use Task Corpus

`tool_use_tasks.jsonl` is generated by `make_tool_use_tasks.py`. The v2
corpus contains 84 benign toy-tool rows: ten rows each for calculator,
dictionary, calendar, file search, route planning, and unit conversion, plus
twenty-four matched no-tool controls.

The dataset is designed to make a surface heuristic a serious baseline without
letting it be the answer key. Many positive rows avoid obvious lexical cues
such as `convert`, `calendar`, or arithmetic operators, while no-tool rows
contain misleading tool names, numbers, filenames, routes, or units. A positive
Lab 34 result should therefore beat both shuffled labels and the deterministic
surface baseline on held-out rows.

Safety scope:

- all tools are closed local simulators;
- no real files, calendars, network calls, credentials, or external tools are
  accessed;
- no autonomous-agent or persistent-goal claim is licensed by this corpus.

Splits:

- positive tool rows: seven train and three eval rows per tool family;
- no-tool controls: sixteen train and eight eval rows.

Known limitations:

- The corpus is still a toy harness, not a real agent benchmark.
- The causal intervention is an action-letter logit test, not open-ended tool
  execution.
- Surface-cue failures should be treated as informative negatives, not hidden
  clutter.
